# OneMarkDotNet 开发指南

## 1. 环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Windows | 10 1809+ | WebView2 需要最低系统版本支持 |
| .NET Framework | 4.8 SDK | 项目目标框架 |
| Visual Studio | 2022 (17.9+) | C# 9.0 语法支持 |
| OneNote | 2016 / Microsoft 365 | COM Add-In 宿主程序 |
| Microsoft Edge WebView2 | Runtime | LaTeX / 图表离屏渲染依赖 |

> **提示**：如果不确定 WebView2 Runtime 是否已安装，可在 `C:\Program Files (x86)\Microsoft\EdgeUpdate` 目录下检查，或访问 `edge://settings/help` 确认 Edge 版本（Chromium 版 Edge 自带 WebView2 Runtime）。

---

## 2. 获取源码

```bash
git clone <repository-url>
cd OneMarkDotNet/src
```

克隆完成后，目录结构应包含 `OneMarkDotNet.sln` 解决方案文件和 `OneMarkDotNet/` 项目目录。

---

## 3. 构建项目

### 3.1 使用 Visual Studio 构建

1. 用 Visual Studio 2022 打开 `OneMarkDotNet.sln`
2. 还原 NuGet 包：右键解决方案 → **还原 NuGet 包**
3. Debug 构建：按 **F5** 或 **Ctrl+Shift+B**
4. Release 构建：在配置管理器中切换为 `Release` 配置后构建

### 3.2 使用命令行构建

```cmd
msbuild OneMarkDotNet.sln /t:Restore,Build /p:Configuration=Debug
```

Release 构建：

```cmd
msbuild OneMarkDotNet.sln /t:Restore,Build /p:Configuration=Release
```

> **注意**：命令行构建需要使用 **Developer Command Prompt for VS 2022**，或确保 `msbuild` 已加入系统 PATH。

## 6. 编码规范

### 6.1 命名约定

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `MarkdownParser` |
| 接口名 | PascalCase + I 前缀 | `IMarkdownExtension` |
| 方法名 | PascalCase | `ConvertBlock` |
| 属性名 | PascalCase | `IsRealtimeRenderEnabled` |
| 私有字段 | camelCase + 下划线前缀 | `_logger`、`_oneNoteApp` |
| 常量 | PascalCase | `MaxRetryCount` |
| 参数 | camelCase | `markdownText` |

### 6.2 命名空间

- 遵循 `OneMarkDotNet` 根命名空间模式
- 使用传统命名空间块（非 file-scoped `namespace X;`）
- 子命名空间按功能划分，如 `OneMarkDotNet.Elements`

### 6.3 COM 互操作规范

- **所有 COM 互操作调用必须包裹在 try-catch 中**，防止 COM 异常导致 Add-In 崩溃
- 使用 `Marshal.ReleaseComObject()` 及时释放 COM 对象
- 避免在 COM 对象上使用 `dynamic` 的多次链式调用

```csharp
try
{
    dynamic pageContent = _oneNoteApp.GetPageContent(pageId);
    // 处理 pageContent...
}
catch (COMException ex)
{
    AppLogger.Instance.Error($"COM 调用失败: {ex.Message}", ex);
}
```

### 6.4 日志记录

- 统一使用 `AppLogger.Instance` 记录日志
- 日志级别：`Debug`、`Info`、`Warning`、`Error`
- 关键操作入口和异常必须记录日志

```csharp
AppLogger.Instance.Info("开始渲染选区");
AppLogger.Instance.Error($"解析失败: {ex.Message}", ex);
```

### 6.5 异步编程

- **避免在 COM 事件处理中使用 `async void`**，可能导致未捕获的异常使 Add-In 崩溃
- COM 事件处理中如需异步操作，使用 `async Task` 并在内部处理所有异常
- WebView2 相关操作使用 `async Task` 模式

## 7. 添加新的 Markdown 扩展

按照以下步骤添加新的 Markdown 语法支持：

### 步骤 1：创建扩展实现

在 `MarkdownExtension.cs` 中创建新的 `IMarkdownExtension` 实现：

```csharp
public class MyCustomExtension : IMarkdownExtension
{
    public void Setup(MarkdownPipelineBuilder pipeline)
    {
        // 注册行内解析器或块解析器
    }

    public void Setup(MarkdownPipeline pipeline, IMarkdownRenderer renderer)
    {
        // 注册渲染器
    }
}
```

### 步骤 2：添加转换逻辑

在 `MarkdownDocument.ConvertBlock` / `ConvertInline` 中添加对应的转换逻辑，将 Markdig AST 节点转换为项目内部的元素模型。

### 步骤 3：添加渲染逻辑

在 `MarkdownRenderer` 中添加对应的渲染逻辑，将元素模型转换为 OneNote 可识别的格式。

### 步骤 4：注册扩展

在 `MarkdownParser` 构造函数中注册新扩展：

```csharp
_builder.Use<MyCustomExtension>();
```

## 8. 添加新的代码语言高亮

按照以下步骤添加新的代码语言语法高亮支持：

### 步骤 1：注册语言名称

在 `CodeHighlightService.cs` 的 `SupportedLanguages` 数组中添加语言名称：

```csharp
public static readonly string[] SupportedLanguages = new[]
{
    "csharp", "python", "javascript", "rust",
    "mylanguage"  // 新增语言
};
```

### 步骤 2：添加别名映射

在 `LanguageAliases` 字典中添加别名映射，使多种写法指向同一语言：

```csharp
public static readonly Dictionary<string, string> LanguageAliases = new()
{
    // 已有映射...
    { "mylang", "mylanguage" },
    { "ml", "mylanguage" }
};
```

### 步骤 3：添加高亮规则

在 `InitializeRuleSets()` 方法中添加该语言的语法高亮规则，定义关键字、注释、字符串等 Token 的匹配模式和样式。

## 9. 添加新的主题变量

按照以下步骤添加新的 CSS 主题变量支持：

### 步骤 1：注册变量名

在 `CssVariableParser.cs` 的 `SupportedVariables` 数组中添加变量名：

```csharp
public static readonly string[] SupportedVariables = new[]
{
    // 已有变量...
    "--my-custom-color"  // 新增变量
};
```

### 步骤 2：添加映射逻辑

在 `CssStyleMapper.cs` 的 `MapCssToOneNoteStyle` 方法中添加 CSS 变量到 OneNote 样式的映射逻辑：

```csharp
case "--my-custom-color":
    oneNoteStyle.CustomColor = value;
    break;
```

### 步骤 3：添加样式属性

在 `OneNoteStyle.cs` 中添加新属性：

```csharp
public string CustomColor { get; set; }
```

> **提示**：添加新变量后，可在 `Themes/` 目录下的 CSS 文件中使用该变量，实现主题定制。
