# 快捷键参考

NoteMark 通过全局键盘钩子（`KeyboardHook`）在 OneNote 窗口中拦截按键，实现 Markdown 渲染、导出和模式切换等快捷操作。本文档列出所有快捷键、Markdown 语法检测模式、Ribbon 按钮以及设置项对快捷键的影响。

## 全局快捷键

以下快捷键仅在 OneNote 窗口为前台窗口时生效。

### 渲染相关

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Enter` | 实时渲染当前行 | 需在设置中启用实时渲染，且当前行内容需匹配 Markdown 语法模式；按键不拦截，正常传递给 OneNote |
| `F5` | 渲染选中区域或当前行 | 如果有选中文本则渲染选区，否则渲染当前行；按键被拦截 |
| `Ctrl+Enter` | 退出源码模式并渲染 | 将当前块从源码模式切换为渲染模式并立即渲染；按键被拦截 |
| `Tab` | 表格/列表内渲染 | 仅在当前行处于表格或列表上下文中时触发渲染；按键不拦截，正常传递给 OneNote |

### 模式切换

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+,` | 切换源码/渲染模式 | 如果当前块在渲染模式则切换为源码模式，反之亦然；按键被拦截 |

### 导入导出

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `F8` | 导出到剪贴板 | 将当前 OneNote 页面内容转换为 Markdown 并复制到剪贴板；按键被拦截 |

## Markdown 语法检测

`MarkdownRenderHandler.ContainsMarkdownSyntax()` 方法逐行检查文本是否包含 Markdown 语法，实时渲染仅在当前行匹配以下模式时触发：

| 模式 | 示例 |
|------|------|
| 标题 | `# Heading`、`## Heading` |
| 无序列表 | `- item`、`* item`、`+ item` |
| 有序列表 | `1. item` |
| 引用 | `> quote` |
| 代码块 | ` ``` ` |
| 水平线 | `---`、`***`、`___` |
| 表格 | `\| col1 \| col2 \|` |
| 加粗 | `**bold**`、`__bold__` |
| 数学公式 | `$$formula$$` |
| 任务列表 | `- [ ] task`、`- [x] done` |
| 链接 | `[text](url)` |
| 图片 | `![alt](url)` |
| 行内代码 | `` `code` `` |

## Ribbon 按钮

除快捷键外，以下功能可通过 OneNote 功能区的 **NoteMark** 选项卡访问：

| 按钮 | 功能 | 对应快捷键 |
|------|------|-----------|
| 渲染 | 渲染当前页面的 Markdown 内容 | F5 |
| 源码模式 | 切换源码/渲染模式 | Ctrl+, |
| 导出剪贴板 | 将 Markdown 导出到剪贴板 | F8 |
| 导入文件 | 从 .md 文件导入 | — |
| 导出文件 | 导出到 .md 文件 | — |
| 关于 | 显示关于对话框 | — |

## 设置影响

以下设置项存储在 `%APPDATA%\NoteMark\settings.json` 中，会影响快捷键和渲染行为。

| 设置项 | 类型 | 默认值 | 影响的快捷键 | 说明 |
|--------|------|--------|-------------|------|
| `IsRealtimeRenderEnabled` | `bool` | `false` | Enter、Tab | 关闭后 Enter 和 Tab 不再触发实时渲染 |
| `IsLineNumberEnabled` | `bool` | `true` | — | 控制代码块是否显示行号 |
| `IsLatexToImage` | `bool` | `true` | — | 控制数学公式是否渲染为图片 |
| `IsSourceModeDefault` | `bool` | `false` | — | 控制新块是否默认为源码模式 |
| `CurrentThemeName` | `string` | `"dark"` | — | 当前使用的主题名称 |

## 自定义快捷键

当前版本的快捷键硬编码在 `KeyboardHook.cs` 中，不支持通过配置文件修改。

### 修改方式

1. 编辑 `src\NoteMark\AddIn\KeyboardHook.cs` 中的 `HookCallbackProcedure` 方法
2. 修改按键判断逻辑，例如：
   - 修改虚拟键码常量（如 `VK_F5` = `0x74`）
   - 修改修饰键组合（`ctrlPressed`、`shiftPressed`、`altPressed`）
   - 添加新的按键事件
3. 重新构建项目并注册 Add-in 后生效

### 虚拟键码参考

`KeyboardHook.cs` 中已定义的虚拟键码常量：

| 常量名 | 值 | 对应按键 |
|--------|-----|---------|
| `VK_F5` | `0x74` | F5 |
| `VK_F8` | `0x77` | F8 |
| `VK_RETURN` | `0x0D` | Enter |
| `VK_TAB` | `0x09` | Tab |
| `VK_OEM_COMMA` | `0xBC` | ,（逗号） |

### 添加新快捷键的步骤

1. 在 `KeyboardHook.cs` 中定义新的虚拟键码常量
2. 在 `HookCallbackProcedure` 方法中添加按键判断逻辑
3. 定义新的事件（如 `NewShortcutRequested`）
4. 在 `NoteMarkAddIn.cs` 中订阅该事件并实现处理逻辑
5. 重新构建并注册
