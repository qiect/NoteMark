# 主题定制指南

OneMarkDotNet 支持通过 CSS 变量自定义渲染样式，包括字体、颜色、间距和行为开关。本文档介绍主题的目录结构、创建方法、支持的变量以及主题生效的内部机制。

## 主题目录

主题文件存放在以下两个位置：

| 位置 | 路径 | 说明 |
|------|------|------|
| 默认主题 | 程序目录下 `Themes\` 文件夹 | 随程序安装，包含内置主题 |
| 用户主题 | `%APPDATA%\OneMarkDotNet\themes\` | 用户自定义主题，同名时覆盖默认主题 |

此外，`__global.css` 文件用于定义全局变量，对所有主题生效。该文件同样可以在上述两个目录中存在，`%APPDATA%` 中的全局变量优先于程序目录中的。

每个主题是一个 `.css` 文件，文件名（不含扩展名）即为主题名。例如 `dark.css` 对应主题名 `dark`。

## 创建主题

1. 在主题目录下创建 `.css` 文件（推荐放在 `%APPDATA%\OneMarkDotNet\themes\`）
2. 在文件中定义 `:root { }` 块并设置 CSS 变量
3. 在 OneNote 的 **OneMarkDotNet** 选项卡中点击 **Theme → Reload** 重新加载主题
4. 从 **Theme** 菜单中选择新主题

## 示例主题

### 暗色主题（dark.css）

```css
:root {
  --font-family: "Segoe UI";
  --bg-color: #1e1e1e;
  --line-height: 1.7;
  --paragraph-margin: 10px;
  --monospace: "Cascadia Code";
  --select-text-bg-color: #2d2d2d;
  --select-text-font-color: #d4d4d4;
  --blockquote-heading-icons: true;
  --enable-heading-in-blockquote: true;
  --enable-code-line-number: true;
  --enable-latex-to-image: true;
  --block-width-margin: #404040;
}
```

### 学术主题（academic.css）

```css
:root {
  --font-family: "Times New Roman";
  --bg-color: #fffff8;
  --line-height: 1.8;
  --paragraph-margin: 12px;
  --monospace: "Courier New";
  --select-text-bg-color: #f9f7f1;
  --select-text-font-color: #333333;
  --blockquote-heading-icons: false;
  --enable-heading-in-blockquote: false;
  --enable-code-line-number: true;
  --enable-latex-to-image: true;
  --block-width-margin: #cccccc;
}
```

## 支持的变量

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--font-family` | 字体名 | `"Segoe UI"` | 正文主字体 |
| `--bg-color` | 颜色值 | `#ffffff` | 页面背景色 |
| `--line-height` | 数值 | `1.6` | 行高倍数 |
| `--paragraph-margin` | 像素值 | `8` | 段落间距（可带 `px` 后缀） |
| `--monospace` | 字体名 | `"Consolas"` | 代码块等宽字体 |
| `--select-text-bg-color` | 颜色值 | `#f5f5f5` | 代码块背景色 |
| `--select-text-font-color` | 颜色值 | `#333333` | 正文前景色 |
| `--blockquote-heading-icons` | `true` / `false` | `false` | 引用块是否显示标题图标 |
| `--enable-heading-in-blockquote` | `true` / `false` | `false` | 是否启用引用块内的标题 |
| `--enable-code-line-number` | `true` / `false` | `true` | 代码块是否显示行号 |
| `--enable-latex-to-image` | `true` / `false` | `true` | LaTeX 公式是否渲染为图片 |
| `--block-width-margin` | 颜色值 | `#cccccc` | 块级元素边框颜色 |

> **注意**：不在此列表中的 CSS 变量会被解析器忽略，不会产生任何效果。

## 主题如何生效

主题从 CSS 文件到最终渲染到 OneNote 页面，经过以下处理流程：

1. **ThemeManager** 从 CSS 文件加载主题并解析变量
   - 读取 `Themes\` 和 `%APPDATA%\OneMarkDotNet\themes\` 目录下的 `.css` 文件
   - 使用 `CssVariableParser.Parse()` 提取 `:root { }` 中的变量
   - 将全局变量与主题变量合并（主题变量优先）

2. **CssStyleMapper** 将 CSS 变量映射为 `OneNoteStyle` 对象
   - `MapCssToOneNoteStyle()` 方法将每个 CSS 变量转换为对应的样式属性
   - 数值型变量（如 `--line-height`）会进行类型转换
   - 布尔型变量（如 `--enable-code-line-number`）会解析为 `bool`

3. **OneNoteXmlConverter.ApplyThemeToElements** 根据元素类型应用样式
   - 遍历所有块级元素，根据元素类型派生不同样式

4. **OneNoteXmlBuilder** 生成包含样式信息的 OneNote XML
   - `CreateStyledT()` 方法将样式属性编码为 HTML `<span>` 标签的 `style` 属性
   - 最终生成符合 OneNote XML Schema 的页面内容

### 不同元素类型的样式应用规则

| 元素类型 | 应用的样式 | 说明 |
|----------|-----------|------|
| HeadingElement | 加粗 + 主字体 + 字号（按级别递减） | H1=24pt, H2=20pt, H3=18pt, H4=16pt, H5=14pt, H6=12pt |
| ParagraphElement | 主字体 + 前景色 + 背景色 | 继承基础样式 |
| CodeBlockElement | 等宽字体（Consolas）+ 代码背景色 + 10pt 字号 | 行号由 `--enable-code-line-number` 控制 |
| TableElement | 继承基础样式 | 表格使用 OneNote 原生 Table 元素 |
| ListElement | 继承基础样式 | 有序列表使用数字前缀，无序列表使用 `•` |
| TaskListElement | 继承基础样式 | 已完成项显示 `[✓]`，未完成项显示 `[ ]` |
| QuoteBlockElement | 斜体 + 缩进 | 嵌套块递归应用样式 |
| MathBlockElement | 特殊字体（Cambria Math） | 渲染方式由 `--enable-latex-to-image` 控制 |
| DiagramBlockElement | 等宽字体 + 10pt 字号 | 显示类型标签 + 源码 |
| HorizontalRuleElement | 无特殊样式 | 使用 Unicode 横线字符 |
| CodeInlineElement | 等宽字体 + 代码背景色 | 行内代码片段 |
| LinkElement | 标准 HTML 链接样式 | `<a href>` 标签 |
| ImageElement | 无文本样式 | `<img>` 标签 |
| MathInlineElement | Cambria Math 字体 | 行内数学公式 |

## 全局变量

`__global.css` 中定义的变量对所有主题生效，用于设置跨主题的通用默认值。

**示例 `__global.css`：**

```css
:root {
  --font-family: "Segoe UI";
  --line-height: 1.6;
  --paragraph-margin: 8px;
  --monospace: "Consolas";
  --enable-code-line-number: true;
  --enable-latex-to-image: true;
}
```

**优先级规则：**

1. 主题文件中的变量值优先于全局变量
2. `%APPDATA%` 中的 `__global.css` 优先于程序目录中的 `__global.css`
3. `%APPDATA%` 中的同名主题文件覆盖程序目录中的同名主题

## 主题数量限制

- 最多支持 **10** 个主题（由 `ThemeManager.MaxThemes` 常量控制）
- 主题按 **最后修改时间倒序** 排列
- 用户主题与默认主题同名时，用户主题覆盖默认主题

## 注意事项

- CSS 文件必须包含 `:root { }` 块，否则变量不会被解析
- 不支持的 CSS 变量会被忽略，不会产生错误
- 主题文件编码应为 **UTF-8**
- 修改主题文件后需要点击 **Theme → Reload** 重新加载才能生效
- CSS 注释（`/* ... */`）会被正确忽略
- 字体名可以用引号包裹（如 `"Segoe UI"`），也可以不包裹
- `--paragraph-margin` 的值可以带 `px` 后缀（如 `10px`），也可以只写数字（如 `10`）
